# 06 - UI / UX Guideline

## หลัก UI สำหรับโปรเจคนี้

กานต้องการระบบที่:

- เรียบง่าย/plain
- เบาและเร็ว
- ใช้ได้บนมือถือพนักงานเครื่องเก่า
- ไม่มี animation/effect หนัก
- DaisyUI 

## Mobile-first Design

### Layout

- 1 หน้าทำ 1 งานหลัก
- ปุ่ม action หลักอยู่ด้านล่างหรือจุดที่กดง่าย
- ใช้ card เรียบ ๆ ไม่ใช้ตารางใหญ่บนมือถือ
- ตัวอักษรต้องอ่านง่าย
- ปุ่มอย่างน้อย 44px height

### Navigation

เมนูหลักมือถือ:

1. หน้าแรก
2. สต็อก
3. รับเข้า
4. แจ้งงาน
5. เบิก
6. Logistics
7. รอตรวจ/ซ่อม
8. รายงาน
9. ตั้งค่า

## Key Screens

### Inventory Search

ต้องมี:
- ค้นหาด้วยชื่อ/item/brand/size/asset tag
- filter warehouse
- filter bucket
- แสดง available/reserved/transit/quarantine
- toggle แสดงรายการ 0

### Receive

Flow:

```text
เลือกคลัง -> scan/เลือก item -> ใส่จำนวน -> สภาพ -> ยืนยัน
```

### Issue

Flow:

```text
เลือก Job -> ระบบโหลด JobLine -> แสดง stock ตามคลังของ Job -> ยืนยันเบิก
```

### Logistics Jobs

Tabs:
- รอรับงาน
- ดำเนินการ
- เสร็จสิ้น

ปุ่มตามสถานะ:
- รับงาน
- ไปหน้าเบิกพัสดุ
- ถึงหน้าร้าน
- บันทึกส่งมอบ/รับคืน
- ต้องเบิกพัสดุก่อน

### Fulfillment

แยก section:
- รายการส่งมอบ
- รายการรับคืน

รายการรับคืนต้องเก็บ:
- สภาพ
- สาเหตุ
- serial/asset tag ถ้ามี
- รูปภาพถ้ามี

### Repair Review

แสดงคิว:
- รอตรวจ
- รอซ่อม
- ซ่อมเสร็จ
- ตีซาก/สูญหาย

ปุ่ม:
- กลับสต็อก
- ส่งซ่อม
- ตีซาก
- สูญหาย

## Error UX

Error ต้องบอกชัด:

ไม่ดี:
```text
Error
```

ดี:
```text
ยังส่งมอบไม่ได้: เบิกพัสดุไม่ครบ
แจ้งส่ง 3 / เบิกแล้ว 1 / คงเหลือต้องเบิก 2
กรุณากลับไปหน้าเบิกพัสดุ
```

## Loading / Offline

- ปุ่มที่กดแล้วต้อง disable กันกดซ้ำ
- ถ้าเน็ตช้าให้แสดง loading สั้น ๆ
- ถ้า offline ใน phase หลัง ให้เก็บ draft queue

## Visual Status Colors

| Status | สี |
|---|---|
| พร้อมใช้ | เขียว |
| จอง | น้ำเงิน |
| ระหว่างส่ง | ฟ้า/คราม |
| รอตรวจ | ม่วง/เหลือง |
| รอซ่อม | ส้ม |
| ตีซาก/สูญหาย | แดง/เทา |
| เสร็จสิ้น | เขียวเข้ม |

## ห้ามทำ

- ห้ามใช้ animation เยอะ
- ห้ามใช้ modal ซ้อน modal
- ห้ามให้ฟอร์มยาวเกินในหน้ามือถือ
- ห้ามแสดง table กว้างบนมือถือ
- ห้ามใช้คำว่า “มือสอง” ในระบบ ให้ใช้ “สต็อก”
